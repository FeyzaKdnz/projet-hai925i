## Description du projet

L'objectif de cette application est de fournir une interface centralisée pour gérer l'inventaire des locaux de plusieurs universités. Elle implémente une architecture Multi-Tenant (Multi-Universités) garantissant une isolation stricte des données entre les établissements (ex: Université de Montpellier vs Université Paul Valéry)

## Fonctionnalités

#### Gestion des Utilisateurs & Sécurité :
* Authentification sécurisée via Spring Security.
* Gestion des rôles : Administrateur (accès total), Enseignant (vue étudiants), Étudiant (lecture seule).
* Suivi de la dernière date de connexion.

#### Gestion des Ressources (CRUD) :
* Gestion des Campus, Bâtiments et Salles.
* Ajout, Modification et Suppression de Salles et d'Utilisateurs.

#### Recherche Avancée :
* Filtres dynamiques pour les salles (par capacité, équipement, bâtiment, étage).
* Filtrage des utilisateurs par rôle et campus.

#### Géolocalisation & Carte Interactive :
* Intégration d'OpenStreetMap (Leaflet) pour visualiser les campus.
* Calcul de distance automatique entre les campus basé sur les coordonnées GPS.

#### Tableau de Bord (Dashboard) :
* Statistiques en temps réel (nombre de salles, répartition par type).

## Stack Technique
* Langage: Java (JDK 17+)
* Framework Backend : Spring Boot 3.x (Web, Data JPA, Security)
* Base de Données : MySQL (ou MariaDB)
* ORM : Hibernate (JPA)
* Frontend : Thymeleaf (Moteur de template), Bootstrap 5 (CSS)
* Outils de Build : Maven
* Cartographie : Leaflet JS / OpenStreetMap

## Installation et Démarrage

#### Prérequis

* Java JDK 17 ou supérieur
* MySQL Server (ou MariaDB)
* Maven

### 1. Configuration de la Base de Données

Créez une base de données vide (ex: campus_db) dans votre SGBD. Assurez-vous que le fichier src/main/resources/application.properties correspond à votre configuration locale

### 2. Lancement de l'application

À la racine du projet, lancez la commande :
``` 
mvn spring-boot:run 
```
L'application sera accessible à l'adresse : http://localhost:8080

## Identifiants de Démonstration (Dataset)
Le projet est pré-chargé avec un jeu de données complet (data.sql) permettant de tester les différents scénarios et universités.

## Architecture du Code

Le projet suit une architecture MVC stricte en couches :

* `src/main/java/com/hai925i/campus/`
    * **`config/`** : Configuration de sécurité (`SecurityConfig`).
    * **`controller/`** : Gestion des requêtes HTTP (`LoginController`, `SalleWebController`, `UserWebController`).
    * **`domain/`** : Interfaces d'accès aux données (`UserRepository`, `SalleRepository`).
    * **`model/`** : Entités JPA (`Universite`, `Campus`, `Salle`, `User`).
    * **`security/`** : Logique d'authentification (`CustomUserDetails`).
    * **`service/`** : Logique métier.
